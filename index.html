How We Work

📅

Weekly Cadence

  • Monday: Week planning + priority sync
  • Wednesday: Mid-week check-in (async)
  • Friday: Demo day + code freeze
  • Async updates in Discord daily
🔍

Code Review Rules

  • All PRs require 1+ review before merge
  • No direct pushes to main branch
  • Tests must pass (when applicable)
  • Clear commit messages required
✅

Quality Bar

  • Ship working software, not perfect code
  • Document as you go (README first)
  • Accessibility is not optional
  • If it's not tested, it's broken
