Contributing

Help us build and improve Atrak projects — from code and design to docs, testing, and feedback.

Ways to Contribute

  • Build: features, fixes, refactors, performance work.
  • Docs: READMEs, setup guides, tutorials, screenshots.
  • Design: UI polish, accessibility improvements, UX flow reviews.
  • Test: phone/desktop testing, bug reproduction, edge-case checks.
  • Ideas: share suggestions or new project proposals.

Getting Started

  • Pick a project from Projects and read its README.
  • If the repo has contribution rules, follow them.
  • Start small: one issue, one improvement, one PR.
  • Include screenshots or a short demo clip when UI changes are involved.

Pull Request Guidelines

  • Keep PRs focused (one clear goal).
  • Match the repo’s code style and patterns.
  • Do not commit secrets (API keys, tokens, private data).
  • If you find a security issue, do not open a public issue — use Security.

Where to Send Feedback

  • Anonymous feedback: use the Suggestion Box.
  • Join / collaborate: use the form on Join.
  • Security reports: use Security.

Thanks for helping us ship real products with real impact.
