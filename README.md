# Atrak

A modern, minimal tech team website + project hub (atrak.dev) featuring a clean startup aesthetic inspired by Vercel and Cursor. Built with pure HTML5, CSS3, and JavaScript.

## ✨ Features

### Design
- 🌑 **Dark Theme**: Professional black background with blue-purple gradient accents
- 🎨 **Glass Morphism**: Modern glassmorphic effects on cards and UI elements
- 💫 **Smooth Animations**: Intersection Observer-based scroll animations and hover effects
- 📱 **Fully Responsive**: Mobile-first design that works on all devices
- ♿ **Accessible**: Semantic HTML with proper ARIA labels

### Sections
- 🏠 **Hero Section**: Eye-catching landing with gradient text and floating card animations
- 📊 **Stats Display**: Animated counter showing key metrics
- 📈 **Impact Analytics**: Auto-aggregated metrics, trend chart, and release milestones
- 🚀 **Projects Showcase**: Notable technical projects with modern card layouts
- 📖 **About Section**: Company information with interactive code preview
- 👥 **Leadership Team**: Team member profiles with social links and personal pages
- 📘 **Team Handbook**: Public handbook covering roles, values, cadence, and quality bar
- 🎯 **Purpose Page**: Mission, values, and what we build
- 📝 **Blog System**: Technical blog posts with views, comments, and content moderation
- 🗓️ **Release Notes**: Monthly updates on what shipped, what broke, and what's next
- 🧾 **Press Kit**: Logos, screenshots, one-pager, and sponsor contact funnel
- 📬 **Anonymous Suggestion Box**: Visitors can send feedback without sharing identity
- 🔗 **Footer**: Comprehensive navigation and information

### Performance
- ⚡ **Optimized Scrolling**: RequestAnimationFrame-based scroll throttling
- 🚀 **Cached DOM Queries**: Minimal reflow and repaint
- 💨 **Smooth Parallax**: Hardware-accelerated CSS transforms
- 🎯 **Lazy Loading**: Intersection Observer for on-demand animations
- 📦 **Split Blog Data**: Preview JSON for fast listing, individual files for full posts

### SEO & Indexing
- 🔍 **SEO Optimized**: Unique meta tags, Open Graph, Twitter Cards on every page
- 📋 **Structured Data**: JSON-LD schema for Organization, WebSite, and projects
- 🗺️ **Sitemap & Robots**: Complete sitemap.xml and robots.txt for search engines
- 🔗 **Internal Linking**: Breadcrumbs and descriptive anchor text throughout

## 🚀 Getting Started

1. **Clone the repository**
   ```bash
   git clone <repo-url>
   cd <repo-folder>
   ```

2. **Open in browser**
   ```bash
   # Using Python
   python3 -m http.server 8080
   
   # Using Node.js
   npx http-server -p 8080
   
   # Or simply open index.html in your browser
   ```

3. **Visit** `http://localhost:8080`

## 📬 Suggestion Box Setup (Anonymous Email)

The site includes an anonymous suggestion form that posts to a backend endpoint (recommended: Formspree) so your email address is not displayed on the page.

1. Create a form at `https://formspree.io` and set the recipient to your inbox.
2. Copy your Formspree endpoint URL (looks like `https://formspree.io/f/xxxxxx`).
3. Update `config.js` and replace `REPLACE_ME` in `window.ATRAK_CONFIG.forms.default` (used by both the Suggestion Box and Join/Contact form).

## 📝 Blog System

The blog uses a split data structure for optimal performance:

- **Preview File**: `data/blog-posts-preview.json` contains metadata (title, excerpt, date, author, tags) without full content
- **Individual Posts**: `data/blog-posts/*.json` contains complete blog post content
- **Features**: View counting, comment system (localStorage), content moderation for inappropriate content

### Adding New Blog Posts

1. Create a new JSON file in `data/blog-posts/` with the slug as filename (e.g., `my-new-post.json`)
2. Add the preview entry to `data/blog-posts-preview.json`
3. The post will automatically appear on the blog listing page

## 👥 Team Profiles

Team member profiles are dynamically generated from `data/team-members.json`. Each member can have:
- Basic info (name, role, bio)
- Social links
- Hobbies and favorite foods
- Personal traits and stats

Access profiles via: `team/profile.html?name=Member%20Name`

## 🧊 GitHub Projects Cache (Avoid Rate Limits)

The "More Projects" tab can load from a cached GitHub snapshot generated by GitHub Actions (`data/github-repos.json`).
This avoids GitHub API rate limiting for visitors and keeps any token private (no token is shipped to the browser).

1. Create a GitHub token (fine-grained PAT recommended) with the minimum read permissions needed for public repository metadata.
2. Add it as a repo secret named `GH_PAT` (Settings → Secrets and variables → Actions).
3. Run the **Update GitHub Data** workflow (Actions tab), or wait for the daily schedule.

The workflow also writes `data/github-meta.json` (timestamp + repo count) for UI display/debugging.
